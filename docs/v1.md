# Daily Calorie Tracker

Vanilla JavaScript로 구현한 **데일리 칼로리 트래커**입니다.  
음식 이름과 탄수화물, 단백질, 지방을 입력하면 음식 카드가 생성되고, 저장된 데이터를 기반으로 총 칼로리와 영양소 차트를 
확인하는 기능들을 기본으로 생각하고 mvp로 빠르게 구현했습니다.

브라우저의 **localStorage**를 사용하여 데이터를 저장하며 음식 **추가 / 수정 / 삭제 기능**을 제공합니다.

---

# 소개

**Vanilla JavaScript 기반 MVP 기능 구현**입니다.

- DOM 조작
- 이벤트 위임
- localStorage 기반 데이터 관리
- 입력 유효성 검사
- 데이터 정규화(normalize)
- Chart.js를 활용한 데이터 시각화

MVP 기능을 중심으로 구성되어 있습니다.

- 음식 데이터 추가
- 음식 데이터 수정
- 음식 데이터 삭제
- 총 칼로리 계산
- 영양소 차트 시각화

---

# 주요 기능

### 음식 추가

- 음식 이름
- 탄수화물
- 단백질
- 지방

입력 후 저장하면 localStorage에 데이터가 저장되고 음식 카드가 화면에 렌더링됩니다.

---

### 음식 수정

카드의 **수정 버튼**을 누르면 수정 가능한 입력 UI로 전환됩니다.

수정 후 저장하면 기존 데이터가 업데이트되고 화면이 다시 렌더링됩니다.

---

### 음식 삭제

카드의 **삭제 버튼**을 누르면 해당 음식 데이터를 삭제합니다.

---

### 총 칼로리 계산

저장된 모든 음식 데이터를 기반으로 총 칼로리를 계산합니다.

칼로리 계산 방식

- 탄수화물 : 1g = 4 kcal
- 단백질 : 1g = 4 kcal
- 지방 : 1g = 9 kcal

---

### 영양소 차트 시각화

저장된 음식 데이터를 기반으로

- 탄수화물
- 단백질
- 지방

총합을 **막대 차트로 시각화**합니다.

Chart.js 라이브러리를 사용했습니다.

---

### 입력 유효성 검사

음식 추가 및 수정 시 다음 조건을 검사합니다.

- 음식 이름은 공백만 입력할 수 없음
- 탄수화물 / 단백질 / 지방 값은 반드시 입력해야 함
- 영양소 값은 숫자여야 함
- 영양소 값은 0 이상이어야 함

# 핵심 구현 방식

## storage 기반 전체 재렌더링

음식 추가 / 수정 / 삭제 후에는 `init()`을 호출하여 전체 데이터를 다시 읽고 화면을 재구성합니다.

이 방식으로 다음 문제를 줄였습니다.

- DOM 직접 조작과 전체 렌더링 혼용
- 영양소 합계 중복 누적
- UI 상태와 저장 데이터 불일치

---

## 이벤트 위임

삭제 / 수정 / 저장 버튼 이벤트는 **foodList에 이벤트 위임 방식**으로 처리했습니다.

이 방식으로 동적으로 생성되는 카드에도 이벤트를 따로 등록할 필요 없이 동작하도록 구성했습니다.

---

## validate + normalize 구조

데이터 저장 전 두 단계의 처리를 거칩니다.

### validateFood()

입력값이 올바른지 검사합니다.

- 빈 문자열 검사
- 숫자 검사
- 음수 검사

### normalizeFood()

저장 가능한 형태로 데이터를 정리합니다.

- 공백 제거
- 숫자 타입 변환
- 데이터 형식 통일

- 사용자별 음식 데이터 관리

- 일간 / 주간 / 월간 칼로리 통계

### 4. Web Components / Lit 라이브러리 실험 (실험 브랜치)

- Custom Element 기반 UI 캡슐화
- 선언적 템플릿 렌더링

해당 실험은 **별도의 실험 브랜치에서 진행**할 예정입니다.

# 회고

Vanilla JavaScript로 MVP 구현하는 과정에서 상태 관리 기준을 DOM에 둘지 storage에 둘지 고민했니다.
초기에는 DOM 중심으로 구현했지만 데이터와 UI 상태가 쉽게 불일치할 수 있다는 문제를 발견했다.
이를 해결하기 위해 storage 기준으로 전체 렌더링하는 구조로 변경하면서 데이터와 UI 상태를 일관되게 유지하는 방법을 이해할 수 있었습니다.
